# Tileworld → Forest Parity Port — Roadmap

> Synthesis of 14 subsystem audits (forest vs. original TS `D:\tileworld\src` vs. ditched bevy port `D:\tileworld-bevy`). Source of truth for sequencing the full-parity port that keeps forest's visuals. Numbers below are preserved verbatim from the audits; where a number is TS-anchored and must be rescaled to forest's combat units, that is called out inline.

## 1. Goal & locked decisions

**The four locked decisions:**

1. **Workspace + vendor `crates/core`.** Restructure forest from a flat single crate into a Cargo **workspace** and add a path-dep (or vendor) `tileworld-bevy/crates/core` (`tileworld_core`). Friction-free: both are **edition 2024**, and `crates/core/Cargo.toml` has an **empty `[dependencies]`** (pure `f64`, no Bevy, no I/O). Build gameplay on top; **KEEP forest's visuals** (scene/terrain/wind/depth-blur/atmosphere are already better than anything in bevy).
2. **Full parity, in dependency order, checkpointing at each runnable build.** Every TS subsystem ends up ported; phases are ordered so each ends at a build that compiles, runs, and is screenshot-able.
3. **Parity bar = "feels the same."** Match mechanics + rough numbers; reuse exact ported logic where it is free (core's pure, test-gated modules); do **NOT** chase byte-exact RNG / hash / map layout.
4. **Cadence = roadmap → mostly-autonomous phased execution**, with a runnable build + summary at each checkpoint.

**Forest's DELIBERATE divergences to PRESERVE (do not "fix"):**

- **World-space**, castle at world origin — NOT the TS grid-space `<group position={-CENTER}>` offset (`CENTER_X=101, CENTER_Z=76`). Drop every `player_world_pos` / `CENTER_X` / `CENTER_Z` / `shift_to_centre` mapping when transcribing bevy code.
- **Ork base HP re-anchored to 60 / 32 / 72 / 47** (grunt/scout/berserker/shaman), vs TS 254 / 136 / 306 / 201 (≈ 0.236×). Keep the ratios; **rescale any TS/bevy/core combat number** (defender bolt damage, guard damage, animal HP, ore HP) into forest's units. Bounty gold/XP are kill-event rewards independent of HP — port as-is, no rescale.
- **Enlarged/relaid map** (`MAP_SCALE 1.4`, 202×151, 5 regions, height base y=0 not y=1). Exact map layout parity is OUT of scope. Forest's mesh stays forest's; only **logic queries** come from core, behind an f32 / −1-Y shim.
- **Bespoke Atmosphere / IBL / colour-grade lighting** — do NOT match TS colour constants. Forest's look is the deliverable.
- **Forest's `siege.rs` wave director is canonical** and richer than bevy's `crates/core/wave.rs`: it adds difficulty mods, `MIN_PREP_SECONDS`, `prep_progress`, `spawn_point`. Do NOT regress to bevy's `wave.rs` (which lacks `MIN_PREP_SECONDS` + difficulty mods and uses TS-scale HP).
- **Keep tuning is forest's**: `KEEP_MAX_HP 1000`, `KEEP_DAMAGE 6`, `KEEP_ATTACK_RANGE 14`, `KEEP_REPAIR_RATE 12/s` (ungated). TS used 650 + 350-reinforced, 6/s gated. Keep forest's.
- **Block stamina in 0..100 units** (forest `38/26/0.6/30`), not TS 0..1 (`0.3/0.18/0.34/...`). Keep forest's units; only add the frontal-cone gate if desired.
- **Depth-blur node** (`depth_blur.rs`) beats both TS bokeh DoF and bevy's `DepthOfField`. Do NOT replace it.

### 1a. Locked execution decisions (resolved with user before P0)

1. **Player model — adopt `core::Player` as `PlayerRes`.** ONE hp + progression home (125 hp, xp/level/gold/crit/lifesteal/cleave/move-speed/bounty/attack-damage). Migrate `HeroHealth`'s live vitals into it so there is no second hp owner; every upgrade/combat effect targets `PlayerRes` directly.
2. **Boot flow — add StartScreen + difficulty chooser + GameOver/Paused.** Full menu parity: `StartScreen` (Enter→Playing), difficulty select (easy/normal/hard, replacing the dev `G` cycle), fresh-run reset on `OnEnter(Playing)`, GameOver + Paused screens.
3. **A* pursuit — pulled forward to P1.** The worldmap **logic-shim** (core `tilemap` queries behind an f32 / −1-Y adapter) + `MapGrid` + 8-dir A* land in **P1** (not P6) so ork/villager/guard pursuit routes around cliffs + house blockers early. Worldmap tasks 2–4 move P6→P1. The visual ground mesh stays forest's; only logic queries come from core.
4. **Defaulted (toward parity; override anytime):** war-bell = diegetic **E** (B kept as debug fallback); shop = merchant-NPC/key interim (not B); defenses **upgrade-gated** (economy lands P3 before defenses P4, so no always-on stub needed); stone source = ore nodes seeded on forest's rock biome; keep self-repair stays forest's ungated 12/s for now (revisit at P4 balance); defender damage rescaled to forest's 60-hp ork units at the call site; rescue long-tail sequenced last (needs the P5 lives pool).

## 3. The structural spine (P0 — everything depends on this)

This is the **biggest and riskiest** change. Forest has **zero** `States` / `init_state` / `FixedUpdate` / `run_if` / freeze-gate anywhere (grep confirmed across all 14 audits). Every gameplay system runs unconditionally in `Update` with ad-hoc `if *mode != PlayMode::Play { return }` guards. Almost every later subsystem must be gated behind the spine. Build it as a sequenced set:

**(a) Cargo workspace + vendor `crates/core`.** Root `Cargo.toml` `[workspace] members = ["."]` (or move forest into `crates/`), add `tileworld_core = { path = "../tileworld-bevy/crates/core" }` (or vendor it in). Both edition 2024, core has empty deps → compiles unchanged. Gate: `cargo test -p tileworld_core` runs core's upgrade_store/shop_catalog/animal/defense/etc. tests in-tree. **Prerequisite for ALL reuse.** Take from core only what each subsystem needs — do **NOT** pull core's `tilemap`/`obstacles`/`house_blockers` as a *second map generator* for movement (forest already has `worldmap`/`steer`/`blockers`); use core's tilemap only for **logic queries** behind a shim, never for the visual mesh.

**(b) `AppState` + `Modal` sub-state + freeze gate.** Reuse `crates/game/src/lib.rs:78-106` **verbatim** (adapt names): `#[derive(States)] AppState { StartScreen, Playing, GameOver, Paused }` + `#[derive(SubStates)] #[source(AppState = Playing)] Modal { None, Shop, UpgradeTree, Inventory }` (+ Paused). The freeze gate is literally `.run_if(in_state(Modal::None))` — the declarative form of TS `isFrozen() = paused || shopOpen || treeOpen || inventoryOpen || settingsOpen`. **Load-bearing trap (documented at `lib.rs:90-106`):** panels MUST be a *sub-state of Playing*, NOT top-level `AppState` variants — otherwise opening one fires `OnExit(Playing)`/`OnEnter(Playing)` and wipes the run. Optionally model `WorldFrozen` as a `ComputedState` over `(AppState, Modal)` (true unless Playing+Modal::None) for one gating condition. Reuse `screens.rs` (StartScreen/GameOver/Paused as `OnEnter/OnExit` bevy_ui overlays, with headless state-machine tests forest can reuse). Reuse `lib.rs:521 pause_toggle` (Esc opens Paused / closes any panel), `lib.rs:543 enter_playing` (the OnEnter(Playing) fresh-run reset), `lib.rs:682 watch_end` (Phase→Victory/Defeat hands off to GameOver, replacing siege's in-`run_director` early-return).

**(c) Fixed timestep.** `insert_resource(Time::<Fixed>::from_hz(60.0))` (`lib.rs:445`); move the sim chain (`player_move`, `player_block`, `player_attack`, siege `run_director`/`invader_brain`, ork/wildlife/villager AI) into `FixedUpdate` `.chain().run_if(in_state(Modal::None)).run_if(hit_stop_inactive)` (`lib.rs:485-501`). Keep camera, anim, particles, day/night, HUD, audio in `Update` (must keep drawing while paused). Replace each system's `if *mode != Play` with the run-condition (keep the FreeRoam check separate). Note: forest movement currently clamps `dt.min(0.05)` (wall-clock); `Fixed` removes the need but also the clamp — re-verify jump arc / move distance after the switch.

**(d) Adopt `tileworld_core::player::Player` as a `PlayerRes` resource.** Forest's player is a `Hero` + `HeroHealth` **component** (hp/max/stamina/block fields only; default 100/100) with **no gold/crit/lifesteal/cleave/move_speed_mult/bounty_mult/attack_damage/xp/level** — the exact fields upgrade & combat effects target. Wrap `Player` as `#[derive(Resource)] PlayerRes(Player)` so `apply_effect`/combat write to the right home for free. Reconcile with `HeroHealth`: **decide one HP home** (keep `HeroHealth` for live vitals + add gold/xp/combat fields, OR migrate hp into `Player`) — a split where maxHP lives in `HeroHealth` but upgrades target `Player` makes MaxHp/Lifesteal upgrades silently no-op. `Player` HP consts (125 max, base dmg 25, +14 HP/+6 dmg per level, XP_FIRST_LEVEL 50, xp_to_next = 50×level) are TS-anchored; keep the *curves*, re-anchor base HP only if intended. Forest currently runs hero at 100 HP / flat 25 dmg — item heal (15–100) & weapon bonuses (11–34) were tuned vs ~100–125 HP so they transfer either way.

**(e) Currency / income loop.** Add gold (on `PlayerRes`, start 30) + a stone bank (`tileworld_core::resource_store::ResourceState`). Wire income: **award gold on ork kill** in `player/combat.rs` (apply `bounty_mult`; per-variant gold 8/6/14/18, xp 20/14/30/34 from `core::ork_config`); **+25 gold on cleared wave** when `tax_office`, **+3 bread/wave** when `farm` — hook the `siege.rs` Wave→Prep transition (`run_director`'s `for action in res.actions { … SetPhase(Prep) … }`). **Stone source:** original mines it in the rock highlands; forest's map diverges, so substitute a feels-the-same source (ore nodes seeded on forest's rock biome, or wave-clear grant). Surface gold + stone in `hud.rs`.

**(f) Keybind remap.** Siege owns **B** (war-bell skip), **G** (cycle difficulty), **R** (restart) — all in `siege.rs::siege_controls`. **P** pauses the day/night clock (`scene.rs`, NOT a real pause). **Esc** releases pointer-lock (`camera.rs:96`). Dev binds: `1-5/0/M` biome switch, `F1` debug panel, `[`/`]` time scrub, backtick free-roam, Space jump (also FreeRoam up — fine, mode-exclusive). Bevy's economy defaults collide: it opens Shop with **B** (clashes with siege bell) and Tree with **U**. **Resolution:** UpgradeTree = **U** (free); Shop = an NPC/merchant interaction or a non-colliding key (E at merchant, or M/T); Inventory = **I** (free); pause = **Esc** (fold the lock-release into the pause transition); interact / war-bell-E / chest = **E**/**F** (both free); quickbar **Q/Z/X/C** (all free); recruit/rescue = **F** (avoid R). Retire dev `1-5` biome switch before the item hotbar lands. Reconcile the **P** double-meaning (clock pause vs real pause) so the phase-driven sun isn't frozen accidentally.

## 6. Open decisions for the user

- **Player model: adopt `PlayerRes(core::Player)` vs extend `HeroHealth`.** Cleanest is `PlayerRes` (gets gold/crit/lifesteal/cleave/move_speed/bounty/xp/level for free, matches every effect target) but creates a second HP home unless you migrate `HeroHealth.hp` in. Pick one HP owner up front — a split silently no-ops MaxHp/Lifesteal upgrades.
- **Hero base stats: re-anchor to core's 125 HP + level/xp, or keep forest's 100 HP / flat 25?** Item heals (15–100) and weapon bonuses (11–34) transfer either way; adopting core gives xp/level "for free" for the PlayerHud.
- **Stone source substitute** (map diverges from TS rock-highland mining): ore nodes seeded on forest's rock biome (recommended — gives the mine verb a home) vs a flat wave-clear stone grant. Either is "feels the same."
- **Keep self-repair: keep ungated 12/s, or gate behind Reinforced Keep + lower to 6/s (TS)?** Forest's ungated 12/s is divergent-but-coherent; combined with towers it could make the keep unkillable — decide alongside defenses balance.
- **Defender damage: rescale ÷~0.24 to forest's 60-HP orks, or keep raw TS (much stronger defenses, ballista one-shots a grunt)?** Override at the call site either way; decide per feel.
- **War bell: diegetic E-interaction (TS parity) vs keep the bare B keybind?** Recommend E entity (B retained as debug fallback) — but it is new entity/swing/SFX work.
- **Defenses always-on vs upgrade-gated.** Without economy, a stub `Defenses` all-true makes early waves trivial. Gate at least Tower Mastery / Keep Archers / Ballista behind the tree, or ship present-but-modest interim.
- **Shop open mechanism: a key (M/T) vs a merchant-NPC E-interaction.** B is taken by siege; bevy used B as a stand-in. NPC interaction is closer to TS but needs the Trader subsystem.
- **How much of the defenses + rescue long-tail to build.** Tower destructibility + ork-targets-defenders changes wave balance; rescue needs a siege lives/garrison pool first. Both can be deferred or staged node-by-node.
- **StartScreen / save scope.** Forest boots into play and has no `Menu` phase or save. Decide whether to add a StartScreen (+ difficulty chooser + fresh-run reset) and any persistence (TS persists difficulty to localStorage), or keep boot-into-play and make StartScreen optional.
- **A* timing.** Pull worldmap A* + `MapGrid` forward (better ork/villager pursuit) or leave mobs on `steer::advance` local steering until P6? Local steering is acceptable for ambient; A* is the big pursuit-quality upgrade.
