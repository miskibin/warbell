# Ork camps — ambient port (design)

Date: 2026-06-07
Project: `D:\tileworld-bevy-forest` (Bevy 0.18.1 static island viewer)

## Goal

Port the TS game's **ork camps** (`OrkCamp.tsx`, `Ork.tsx`, `CampCage.tsx`, `Tent.tsx`,
`Campfire.tsx`) into the Bevy scene as an **ambient diorama layer** — the same role wildlife
and the castle play. The scene is a viewer: **no combat, HP, damage, death, rescue, respawn,
faction fighting, or player aggro.** Camps are set-dressing that *feels alive*: a guarded
warband mills around tents, a fire, a banner, skull-spikes and a prisoner cage.

Decided with the user:
- **Ambient & alive** orks (patrol/idle, anchored to camp, never roam the island, never flee).
- **5–6 camps** — one per wilderness biome (snow, desert, forest, swamp, rock). Less faithful
  to the TS 3-camp placement, scaled for the enlarged/denser map.
- **All 4 ork variants** (grunt / scout / berserker / shaman) as models.

Camps render in the **Combined world-map view only** (same as the castle). Single-biome views
(keys 1–5) are abstract full-patch scenes with no island layout, so no camps there.

## Architecture

Three new modules + small edits to existing files.

### `steer.rs` (new) — shared local obstacle steering

Factor the steering currently inline in `wildlife::animal_brain` into a reusable unit so
orks get the **same anti-flicker behaviour** (the escape-fan + continuity bias + turn-rate
cap that fixed the 180°-snap jitter). Pure functions, no ECS:

- `pub const MAX_STEP: f32` (0.6) — max ground-height delta per step.
- `can_stand(x, z, r, cur_y) -> bool` — centre + 4 cardinal footprint edges on land within `MAX_STEP`.
- `step_clear(pos, dir, dist, body_r, cur_y) -> bool` — `can_stand` at the step + lead point, and not `blockers::is_blocked`.
- `wrap_pi(a) -> f32`.
- `pub struct Step { pub facing: f32, pub pos: Vec2, pub moving: bool }`
- `pub fn advance(pos, facing, goal, step_dist, body_r, cur_y, max_turn_dt) -> Option<Step>`
  — the escape-fan over `[0, ±0.4, ±0.8, ±1.2, ±1.7]` scored by `dir·goal + 0.6·dir·cur_dir`,
  turn toward the winner clamped to `±max_turn_dt`, then walk along the slewed facing if clear
  (else pivot in place). `None` = boxed in.

`wildlife.rs` is refactored to call `steer::advance` (no behaviour change — same constants).

### `critters.rs` (edit) — biped part kind

Add `PartKind::Arm(f32)` (sign = swing phase, like `Leg`). Wildlife models never use it;
`animal_limbs` adds a no-op arm so its match stays exhaustive.

### `orks.rs` (new) — ork models + ambient AI

**Models** (`build(variant, faction) -> OrkSpec`), ported 1:1 from `Ork.tsx` mesh tree, but
as merged, flat-shaded, **vertex-coloured** meshes against the shared white material (like
`critters`/scatter — batches to few draw calls). No runtime tinting (no combat) — the variant
skin + faction colour are baked at build.

- `enum OrkVariant { Grunt, Scout, Berserker, Shaman }` with per-variant skin + scale
  (grunt green `#3a6a2a` 1.0, scout `#5f9a3c` 0.78, berserker `#7a3a26` 1.06, shaman `#6a3f86` 0.96).
- `enum Faction { Red, Blue }` → loincloth/war-paint/banner/tent tint (`#9a2a22` / `#26468f`).
- Topology (root + children): static **torso** (legs + loincloth + belt + torso + war-paint),
  **Head** (skull + brow + eyes + tusks + ears), **Arm**×2 (shoulder + upper + forearm; the
  right arm's child weapon is **club** for melee variants / **staff + glowing orb** for shaman).
  Legs are part of the static torso? No — give **Leg**×2 as articulated parts so they stride.
  → parts = 2×Leg, 2×Arm, 1×Head. Weapon baked into the right-arm mesh.
- Feet authored at `y≈0` so the root, placed on the ground, plants the ork.

**Component** `Ork { home: Vec2, pos, facing, speed, wander_r, gait, swing, bob, body_r,
phase, moving, mode, timer, rng }` and `OrkPart { kind: PartKind }`.

**Systems** (`OrksPlugin`, `Update`):
- `ork_brain` — `Idle ↔ Patrol` around `home` (camp centre), `wander_r ≈ 3.5`, slow lumber
  speeds, **no camera flee**. Uses `steer::advance`; ground-follows; small bob while moving.
- `ork_limbs` — `Leg(sign)` stride opposite phase; `Arm(sign)` swing opposite the legs (idle
  sway otherwise); `Head` idle scan. Same sin-wave trick as `animal_limbs`.

### `camps.rs` (new) — placement, props, build, reservation

**Placement** — `plan()` deterministically reject-samples **one camp per wilderness biome**
into a `OnceLock<Vec<CampSite>>` (mirrors the `worldmap::TILES` cache). For each biome:
sample a candidate `(cx, cz)` over the island; accept if the **7×7 tile footprint** is all
walkable + all the same (flat) height, not in `castle::in_footprint`, far enough from origin
and from already-placed camps, and the centre is either *in* the target biome **or** on grass
within ~8 tiles of it (so mountainous snow/rock camps land on the flat apron at the biome edge).
`CampSite { centre: Vec2, biome, faction, rot, seed }`. Faction alternates per camp.

- `pub fn in_clearing(wx, wz) -> bool` — within any camp's reserved footprint (reads the
  `OnceLock`). Used by scatter + wildlife to keep their stuff out of camps.

**Props** (vertex-coloured, flat-shaded, `BiomeEntity`-tagged) per `CampSite`, placed at
camp-local offsets rotated by `rot`:
- 2 **tents** (ridge-tent: triangular-prism canvas + door + ridge pole), faction-tinted canvas.
- **campfire** (stone ring + crossed logs + emissive flame cones + rising smoke). Emissive
  flame only — **no point light** (matches castle torches; avoids the point-light-count stutter;
  daytime anyway). A `Flicker` component + system scales the flame; smoke reuses the castle's
  drift pattern (small local copy).
- **banner** (pole + faction flag).
- 2 **skull-spikes** (short pole + skull box).
- **cage** (`CampCage.tsx` port): plank floor, 4 corner posts, top rails, vertical bars on 3
  sides, a **shut** door, 2 huddled captive figures inside (drab prisoner tones). No open/rescue.

Every solid prop registers its tile(s) in `crate::blockers` so orks + wildlife route around them.

**Build** — `pub fn build(commands, meshes, materials)` iterates the planned sites: spawn props
(register blockers) + the warband (`orks::spawn` at local offsets), all `BiomeEntity`.

Warband per camp: **grunt + scout + berserker + shaman** at 4 local offsets around the fire,
`home` = camp centre, `wander_r ≈ 3.5`.

## Integration — `worldmap::build` (Combined view only)

New call order:

1. terrain mesh
2. sea
3. **`camps::plan()`** — *before* scatter, so clearings can be reserved.
4. scatter: each biome predicate becomes `tile_biome_world == Some(b) && !camps::in_clearing(x,z)`;
   the grass-cover predicate gains `&& !camps::in_clearing(x,z)`.
5. `castle::build`
6. **`camps::build()`** — props + orks; registers camp blockers. *Before* wildlife so wildlife
   avoids camp props.
7. `wildlife::populate` — `valid()` gains `&& !camps::in_clearing(x,z)` so no deer in a cage.

`main.rs`: add `mod steer; mod orks; mod camps;` and the `OrksPlugin` + `CampsPlugin`.

Biome switch already despawns `BiomeEntity` + `blockers::reset()` and re-runs `worldmap::build`
for Combined, so camps rebuild cleanly each switch; the `OnceLock` site plan persists (pure
tile-map function), blockers re-register in `camps::build`.

## Non-goals

No rescue/free, no respawn, no rival-faction combat, no player aggro, no HP bars, no day/night,
no camp audio (could be a later pass). Pure ambient set-dressing.
